Infraestrutura como Código (Infrastructure as Code) Controle de Versão Definições

O que é Infraestrutura como Código (IaC)?

Infraestrutura como código (IaC) é o gerenciamento e provisionamento da infraestrutura por meio de códigos, em vez de processos manuais.

Com a IaC, são criados arquivos de configuração que incluem as especificações da sua infraestrutura, facilitando a edição e a distribuição de configurações. Ela também assegura o provisionamento do mesmo ambiente todas as vezes.

Controle de Versão

O controle de versão é uma parte importante da IaC. Os arquivos de configuração devem pertencer à fonte como qualquer outro código-fonte de software. Ao implantar a infraestrutura como código, também é possível separá-la em módulos, que podem ser combinados de diferentes maneiras por meio da automação.

Definições

  • Atualizar Servidor:

    Realizar a atualização do servidor para garantir o seu funcionamento com as últimas correções e melhorias disponíveis.

  • Instalar apache2:

    Configurar e instalar o servidor web Apache (apache2) como parte do ambiente de hospedagem.

  • Instalar unzip:

    Instalar a ferramenta unzip para extrair arquivos compactados da aplicação web, facilitando o processo de instalação e configuração.

  • Baixar Aplicação Web:

    Obter a aplicação web necessária para o servidor web a partir do seguinte endereço: https://github.com/T-Amparo/DIO-DIGITAL-INNOVATION-ONE/archive/refs/heads/main.zip

  • Salvar Aplicação Web:

    Salvar a aplicação web baixada no diretório /tmp.

  • Mover Aplicação Web:

    Após o download, mover os arquivos da aplicação web para o diretório padrão do Apache: /var/www/html/, onde serão servidos para os clientes.
